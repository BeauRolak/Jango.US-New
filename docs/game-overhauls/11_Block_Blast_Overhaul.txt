JANGO.US GAME OVERHAUL FILE - BLOCK BLAST

CLAUDE / AI DEV DIRECTIVE
This file is for one game only. Do not try to fix the entire platform from this file. Use it as the game-specific rebuild/overhaul spec for Jango.US. The goal is to make this game feel premium, alive, fair, reactive, visually strong, and playable from start to finish.

Jango.US product standard:
- Dark near-black competitive arena UI.
- Blue/purple neon identity with each game having its own accent color.
- Scalps economy shown in UI where relevant.
- 3% platform rake shown in match/tournament economics where relevant.
- No real payment movement unless explicitly approved later. UI can show mock/gated economy states.
- No emoji primary icons. Use real icons, custom SVG, or original poster art.
- Every action should feel responsive through motion, sound, haptics, and toast feedback.
- Mobile must feel intentionally designed, not squeezed desktop.

GAME MISSION
Turn Block Blast into a polished puzzle/arcade duel with clear objectives, satisfying block placement, combo animations, bot/ghost comparison, and reliable scoring.

VISUAL / ART / LIFE OVERHAUL
- Use a neon puzzle grid with glowing blocks, particle bursts on line clears, combo shockwaves, and dark arcade backdrop.
- Blocks should have depth, bevels, shadows, and color identity.
- Line clear should be a satisfying horizontal/vertical glow sweep.
- Combo streaks should build visible meter energy.
- Lobby art should show blocks exploding in a neon grid, not generic squares.

CORE GAMEPLAY / MECHANICS OVERHAUL
- Define exact V1 mode: players place block pieces into a grid to clear rows/columns and score within equal turns or time.
- Both players should receive the same piece sequence in competitive matches for fairness.
- Use matchSeed to generate identical piece sequence for human and bot/opponent.
- Score: placement points + line clears + combos + streaks.
- Game ends when no legal move or time/turn limit ends.
- Score only updates after a piece is placed and line clear resolution completes.

MATCH SETUP / SCALPS / ECONOMY UI
- Entry modal: turn-count or timed mode, bot difficulty, entry Scalps, pot, 3% rake, payout.
- Show fairness note: same piece sequence for both players.

BOT SYSTEM - EASY / MEDIUM / HARD
- Easy bot: chooses first valid placements, misses combos.
- Medium bot: maximizes immediate clears and avoids obvious dead spaces.
- Hard bot: looks ahead several pieces, preserves board space, maximizes combos.
- Bot uses same piece sequence and legal placement rules.

WIN / LOSS / REWARD ANIMATIONS
- Combo clear: grid flash and score multiplier pop.
- Final win: board explodes into neon particles, winner score comparison, payout panel.
- Comeback bonus animation if player wins on final placement.

SHARED GAME ARCHITECTURE RULES
Every game should plug into the same reliable match framework:
MATCH_SETUP -> MATCH_START -> ROUND_START -> PLAYER_ACTION -> ACTION_RESOLVED -> SCORE_LOCKED -> NEXT_TURN_OR_ROUND -> MATCH_END -> RESULTS_SCREEN

Hard rules:
- UI cannot directly mutate score.
- Scoreboard only displays confirmed state.
- Bots take one valid action per turn, then stop.
- No bot can loop forever or spam actions.
- No game can stall after a bot turn.
- No game resets state unless it is a legal new round, restart, out-of-bounds, foul reset, or match reset.
- Every match must reach a winner/results screen.
- Every player action must create a unique actionId/shotId/moveId so duplicate scoring cannot happen.
- Every important action should trigger the global feedback system: visual effect + toast + sound + haptic if enabled.

IMPLEMENTATION ORDER
1. Create BlockBlastEngine with grid, pieces, legal placement, clear resolver, and score.
2. Create seeded piece generator.
3. Build bot strategy after engine tests.
4. Add mobile drag/drop with snap preview.

ACCEPTANCE TESTS
- Game loads without console errors.
- Human vs bot match starts correctly.
- Easy, Medium, and Hard bots all complete full matches.
- Score updates only from confirmed actions.
- Score does not duplicate, loop, or climb by itself.
- Game progresses after bot action.
- Winner/result screen appears at match end.
- Restart returns to a clean match state.
- Mobile controls are touch-friendly and do not overflow.
- All major actions trigger sound/haptic/toast feedback through the global system.
- No primary emoji icons are used.
- Vercel build is green and PROGRESS.md is updated.

FINAL NOTE TO CLAUDE
Do not call this game complete until it is playable, visually alive, mobile-friendly, and reaches a clean winner screen. If a choice exists between more polish and more reliability, build reliability first, then add art/motion. The final result should feel like a premium Jango.US game, not a placeholder.
